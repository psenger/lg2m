# References for Layer 6 — `gen` fidelity

Exact wiring this layer reuses. Do not re-derive these; import, read, and call them. File/line cited from
the Phase 1 exploration; verify before editing.

## The forward transform to invert (read as the spec)

- `src/lg2m/diff/assemble.py` — `canonicalize(diagram, edge_kinds, *, file)` (≈ lines 182–256) and the
  helpers `_collapse_pseudostate` (fork/join splice, ≈ 288–295), `_flatten_composite` /
  `_composite_entry_exit` (≈ 212–230), and `START_ID`/`END_ID`, `_CONDITIONAL_KINDS`,
  `_PARALLEL_KINDS` (≈ 39–47). Rule A collapses `<<fork>>`/`<<join>>`; Rule B flattens single-entry/
  single-exit composites to `parent:child`; Rule C maps top-level `[*]` → sentinels. `decanonicalize`
  is the mirror.
- `src/lg2m/diff/assemble.py` — `assemble_doc_model(doc, *, file) -> GraphModel(origin="markdown")` and
  `assemble_code_model(topology, registry, locations) -> GraphModel(origin="code")` (the round-trip oracles).

## IR (identity is structural)

- `src/lg2m/ir.py` — `Node` (id is identity; `is_subgraph`, `meta`, `kind` carried; **no `parent` field**),
  `Edge` (identity `(src_id, dst_id, predicate)`; `conditional`/`is_else`/`parallel` carried; **no `scope`
  field** — scope lives on `MermaidEdge` only), `Route`, `Predicate`, `DataModel`, `Attribute`,
  `GraphModel`, `NodeKind {NODE, START, END}`, `ELSE_LABEL = "[else]"`.

## Mermaid emit primitives (reuse; do not hand-roll)

- `src/lg2m/parsing/mermaid.py` — `emit_mermaid(diagram) -> list[str]` (≈ 104–122) already renders
  `MermaidState(pseudostate=...)` and composite `state X { ... }` by grouping edges on `MermaidEdge.scope`.
  Build `MermaidDiagram(states, edges)`, `MermaidState(id, is_subgraph, pseudostate, parent)`,
  `MermaidEdge(src, dst, predicate, conditional, is_else, scope)`. Note: `MermaidState.parent` is tracked
  but **not used** during emit; scope on edges drives composite blocks.
- `src/lg2m/parsing/tables.py` — `emit_table(headers, rows) -> list[str]` (GFM, `\|` escaping) for the
  Index / Data Models / Edges tables.

## Code-side IR (the `--from-code` input; reuses the `check` chain)
